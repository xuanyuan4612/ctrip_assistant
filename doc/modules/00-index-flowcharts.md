### 端到端闭环链路

1. **主动发现 → 立案调查 → 处罚整改**: 风险监控发现异常 → 推送廉洁监察立案 → 调查取证 → 追责处罚 → 问题汇入持续改善 → 整改闭环
2. **定期审计 → 缺陷发现 → 整改跟踪**: 内控评价/专项审计/离任审计执行 → 发现问题清单 → 汇入持续改善 → 下发整改 → 复核归档
3. **定密管理 → 制度支撑 → 持续优化**: 商业秘密定密 → 调用内控制度库 → 发现制度缺失 → 汇入持续改善 → 制度完善
4. **行为风险 → 分析预警 → 分流处置**: 行为数据分析 → 风险预警 → 推送相关模块(廉洁监察/离任审计) → 处置闭环

### 3.1 廉洁监察流程图（6阶段工作流）

```
案件录入（双轨来源）
    ├── 系统抓取：智能体从风控系统自动抓取举报案件信息
    └── 人工录入：风控人员手动新增案件信息
    ↓
[阶段一] 材料初判 + 分流决策
    ├── 语音转文字（多模态预处理）
    ├── 材料初判报告（是否调查 + 理由 + 处置方式）
    ├── 碳基守门：是否处理？
    │   ├── 否 → Output1: JSON推送风控系统闭环
    │   └── 是 → 是否转交？
    │       ├── 否 → 继续调查
    │       └── 是 → 是否归属HR？
    │           ├── 是 → A2A龟宝承接 → Output2闭环
    │           └── 否 → 任务中心转交其他部门 → Output2闭环
    ↓
[阶段二] 调查方案生成
    ├── 智能信息提取与匹配（关键字段匹配：访谈人员/系统数据/类似案件）
    └── 调查方案生成（初步走向预判 + 调查方案模板.xlsx）
    ↓
[阶段三] 多维分析 + 案件报告撰写
    ├── 多维数据碰撞（数据中台 + 业务系统 + SQL分析）
    ├── 访谈提纲生成 + 访谈记录沉淀（语音转文字）
    ├── 现场走访分析
    ├── 形成案件结论（多维度定性结论报告）
    └── 撰写廉洁监察报告（标准制式报告 + 全量附件）
    ↓
[阶段四] 处置分流 + 追责确定
    ├── 是否涉及追责处罚？
    │   ├── 否 → Output3: JSON推送风控系统闭环
    │   └── 是 → 未来处置走向分析报告
    │       ├── 刑事立案 → 撰写报案书 → 移交司法程序
    │       ├── 民事纠纷 → 推送西塞罗任务中心 → 法务处理
    │       └── 内部处理 → 内部追责初步意见 + 处罚人员清单
    ↓
[阶段五] 处罚执行 + 跟踪
    ├── 撰写处罚公告 → 添可: OA审批 / 集团+科沃斯: 手工上传
    ├── 撰写赔偿协议 → 推送西塞罗审核
    ├── 黑名单维护 → MDM系统
    ├── 供应商扣款跟踪 → 推送波特
    └── 员工扣款跟踪 → A2A龟宝
    ↓
[阶段六] 报案后续协助
    └── 资料提取指引：外部问题清单 → 去哪取数据/如何回答
    ↓
闭环
```

### 3.2 风险监控流程图（6阶段工作流）

```
[6.1] 风险规则清单生成
    输入: 风险规则（事业部/渠道/业态/部门/岗位/一~三级场景）+
          数据库字段及含义 + 历史判例 + 人工自定义场景(可选)
    逻辑: 选择模式(清单上传/人工录入)
          → AI生成三级场景 → 计算规则 → SQL语句
          → AI在测试环境验证SQL可执行
          → 人工审核通过→入库 / 不通过→修正/作废
          → 形成风险清单知识库
    输出: 风险清单知识库（标准15字段）
    ↓
[6.2] 风险异常数据生成
    输入: 风险清单知识库 + 业务数据 + 外部数据
    逻辑: AI按SQL与监控频率自动执行 → 生成异常明细
          → AI初核: 剔除正常业务、标注异常说明
          → 人工二次复核: 通过→进入分析 / 不通过→调整规则/作废
    输出: 风险预警异常数据明细及透视表 + 异常数据分析报告
    ↓
[6.3] 风险分析与主体合并
    输入: 异常风险明细 + 异常风险明细透视表
    逻辑: AI识别分析主体(联系方式/联系人/地址等)
          → 同一主体多条异常自动合并，标记涉及指标
          → 识别风险行为、风险业务、影响范围、涉及金额
    输出: 按分析主体合并后的风险透视表 + 单主体风险分析报告
    ↓
[6.4] 风险自动定性
    输入: 单主体风险分析报告 + 历史审计案例知识库
    逻辑: AI自动判定风险类型(合规/舞弊/商业秘密等)
          + 风险等级 + 严重性 + 广泛性
          + 影响程度(岗位/金额/业务范围/发生频次综合评估)
          + 处置建议
    输出: 风险定性报告及处理建议
    ↓
[6.5] 风险结果自动推送
    推送对象: 业务部门智能体 / 廉洁监察模块 / 内控内审模块 / 商业秘密模块
    输出: 推送记录
    ↓
[6.6] 处置结果回流与指标迭代（闭环核心）
    输入: 各模块处置结果 + 整改情况
    逻辑: 记录状态(已处置/整改中/无需处置/误报)
          → 误报/失效指标 → 人工确认 → 优化SQL/阈值
          → 更新风险清单知识库
    输出: 风险处置闭环台账 + 指标优化迭代记录
```

### 3.3 内控评价流程图（13步工作流）

```
[7.1] 立项
    风控系统立项 → 填写项目基本信息 → 同步至智能体
    ↓
[7.2] 审计方案生成（5部分）
    ① 项目基本信息（项目名称/审计目的/被审计单位/审计期间/组长/组员）
    ② 评价依据（标准内容，调用方案模板）
    ③ 审计范围（从风控系统同步"标准控制活动"底表的业务循环和一级业务流程）
    ④ 审计实施细则（各阶段安排 + 各业务循环实施细则：审计目标/关键控制点/测试程序/时间安排/人员分工）
    ⑤ 缺陷认定标准（标准内容，调用方案模板）
    ↓
[7.3] 生成访谈人员清单及访谈计划
    AI结合部门信息、员工信息、西塞罗岗位职责 → 访谈对象建议 → 制定详细到日的访谈计划
    ↓
[7.4] 生成访谈问卷
    通用部分（所有人涉及）+ 专业部分（根据风险场景/控制活动/岗位职责按问题库生成）
    ↓
[7.5] 根据访谈问卷完善风控矩阵
    访谈纪要 → 结合制度清单、系统清单 → 完善风控矩阵
    ↓
[7.6] 拆分风控矩阵
    汇总表格 → 拆分(一行变多行) → 设计测试底稿 + 执行测试底稿
    ↓
[7.7] 评估内控设计缺陷
    设计测试底稿 → 阅遍相关制度文档 + 历史设计缺陷 + 设计缺陷类型 + 流程设置情况
    → 按评分标准打分 → 输出设计缺陷类型、缺陷描述
    ↓
[7.8] 生成资料需求清单
    执行测试底稿 → AI整合避免重复 → 输出资料需求对接部门/对接人员/样本量/样本期间
    ↓
[7.9] 评估内控执行缺陷
    系统对接数据/手动上传数据 + 流程测试数据 + 历史执行缺陷 + 执行缺陷类型 + 评分标准
    → 输出缺陷类型、缺陷描述
    ↓
[7.10] 内控缺陷输出及确认
    ① 业务初步确认：设计缺陷+执行缺陷清单与业务部门初步沟通
    ② 输出缺陷汇总及沟通确认表：按业务循环/缺陷类型/一阶管理部门汇总统计
    ③ 与业务负责人确认
    ↓
[7.11] 内控评价总体打分
    经确认的缺陷清单 → 对各控制事项打分 → 按事业部/循环维度计算总分 → 得分汇总表
    ↓
[7.12] 出具内控评价报告（5章结构）
    一、项目概述 | 二、内部控制整体结论 | 三、内部控制运行情况
    四、重要缺陷列示 | 五、持续改善推进
```

### 3.4 专项审计流程图（5阶段工作流）

```
（一）审计方案生成
    输入: 审计目的/审计重点/审计区间/被审计单位/组长/组员(风控系统立项)
          + 审计方案库/审计记录库(知识库)
    匹配逻辑: 检索与审计目的/重点关联的历史方案 → 获取历史检查记录 → 综合输出优化方案
    抽样方案: AI根据业务量级和风险程度建议样本量和抽样方法
    输出: 《专项审计实施方案》（审计目的/期间/重点/方法/抽样方案）
    ↓
（二）访谈作业
    输入: 审计方案 + 公司人员岗位职责 + 访谈问题库(知识库) + 问题清单(非必选)
    步骤:
      ① 根据需访谈内容 → 调用人事MCP匹配访谈人员 → 输出访谈计划 → 守门
      ② 根据访谈计划生成访谈问卷 → 守门 → ATA下发各被访谈人员
      ③ 线下访谈后上传文字记录 → AI分析结论完整性 → 判断是否补充提问
    输出: 访谈计划(访谈清单+访谈策略) + 访谈问卷 + 访谈结论
    ↓
（三）检查作业
    输入: 审计方案 + 标准资料需求模板库(知识库) + 访谈结果(非必选) + 资料检查结果(非必选)
    步骤:
      ① 根据业务重点 + 资料清单 + 业务流程库 → 输出资料检查计划 → 守门
      ② AI自动生成SQL从数据中台获取数据 → 按检查点检查问题 → 生成审计底稿/问题清单
      ③ AI汇总分析是否达成审计目的 → 若无法达成则提出资料补充建议及下一步方向
    输出: 资料检查计划 + 审计底稿 + 问题清单
    ↓
（四）问题确认
    输入: 审计发现
    步骤:
      ① 将问题清单拆分 → ATA下发各负责人确认
      ② 反馈确认 → 进入下一步
         反馈不确认 → AI分析反馈是否合理 → 合理:关闭并保存至知识库 / 不合理:输出理由继续下发
    输出: 经确认的审计问题
    ↓
（五）出具审计报告
    输入: 项目基本资料 + 问题清单
    输出: 整改建议 + 审计报告
```

### 3.5 离任审计流程图（6阶段工作流）

```
（一）审计方案生成
    输入: 
      ① 被审计人基础信息(风控系统立项审批后推送): 事业部/姓名/工号/岗位/部门/入职日期/最后工作日/组长/组员/项目名称编号/审计期间
      ② 职责配置库(MCP数据调用): 人员具体职责内容
      ③ 审计方案库(知识库): 个人审计方案库 + 业务审计方案库
    匹配逻辑:
      - 读取被审计人岗位职责列表 → 检索业务审计方案库强关联业务 → 检索个人审计方案库通用/特有检查项
      - 审计期间: 1年<本岗位≤5年 → 离职日期前3年; >5年 → 前1年
      - 抽样方案: AI根据业务量级和风险程度建议
    输出: 《离任审计实施方案》(被审计人信息/审计期间/审计范围/审计方法/抽样方案)
    ↓
（二）生成访谈问卷
    输入: 审计方案 + 被审计人岗位职责 + 访谈问卷通用问题(知识库)
    新增需求: 离职人是否可确认选项 + 下拉框选择审计人员(组长/组员)确认问卷
    输出: 结构化《离任审计访谈问卷》
    ↓
（三）生成资料清单
    输入: 审计方案 + 标准资料需求模板库(知识库)
    新增需求: 资料需求增加提供资料人员名称 → 同一人员多条资料合并发送至任务中心
    输出: 资料需求清单
    ↓
（四）问题清单生成
    输入: 天眼查外部数据 + 业务系统数据(BPM/OA/DTC等) + 人工上传线下数据
          + 行为风险预警(其他模块传回) + 历史个人/业务审计问题
    输出:
      - 个人问题清单: 商业秘密泄露/个人报销/样机使用/关联公司/资产使用
      - 业务问题清单: 流程漏洞/制度缺陷/经济损失
    ↓
（五）审计问题确认
    ① 问题清单明细汇总 → 风控人员守门确认 → 输出最终版缺陷清单
    ② 与业务部门初步沟通确认(发布任务/邮件)
    ③ 生成审计意见表(审计发现问题+审计建议+审计沟通对象) → 推送风控系统
    ↓
（六）出具审计报告
    输入: 已确认的个人问题清单 + 业务问题清单 + 审计报告模板
    输出: 
      ① 标准化《离任审计报告书》(经守门后推送风控系统)
      ② 传递给"整改跟踪智能体"的结构化问题列表
```

### 3.6 商业秘密 + 行为风险流程图

### 3.7 持续改善流程图（全流程闭环）

```
审计问题来源（多模块汇入）:
  内控评价 / 廉洁监察 / 商业秘密 / 行为风险 / 专项审计 / 专项检查 / 业务交办
  ↓
问题录入:
  - 风控系统自动抓取
  - 手动单条录入
  - Excel批量导入(按固定表头模板)
  - 支持问题作废(需填写原因，留痕)
  ↓
  [问题待推送]: 问题清单同步至整改智能体 → 给组长创建"问题推送任务"
  ↓
任务下发:
  组长确认问题录入无误 → 一键"下发整改任务"
  → 系统自动推送至对应部门负责人(任务中心)
  → 同时推送审计跟进人"计划/方案审核任务"
  → 状态更新为 [计划待提交] → Elink通知部门负责人
  ↓
计划提交:
  部门负责人在任务中心提交:
  - 具体责任整改人
  - 审计整改计划/方案
  → 状态更新为 [计划待审核]
  ↓
计划审核:
  AI初审: AI对附件进行初审并给出意见
  → 审计整改跟进人二次确认
  → 通过 → 赫尔墨斯再推送"审计整改任务"(AI自动生成整改目标和任务描述)
  → 状态更新为 [整改答复待提交] → Elink通知责任人
  → 不通过 → 退回(备注原因) → 状态回退 [计划待提交]
  ↓
整改执行:
  责任人在任务中心提交审计整改证据
  (须严格按照已提交的整改计划/方案执行)
  ↓
复核闭环:
  责任人提交整改完成 → 状态更新为 [已整改待复核]
  → AI初审核: 依据前期整改计划对附件进行初审判断
  → AI通过 → 自动向审计岗Elink发送复核提醒 + 向任务平台推送复核任务
  → 审计岗复核:
      ├── 复核通过 → 状态更新为 [整改完成] → 自动归档(不可编辑)
      └── 复核不通过 → 填写退回原因 → 状态更新为 [退回重改]
          → 向责任人发送退回提醒
          → 退回次数≥3次 → 同步通知所在部门负责人
  ↓
归档:
  整改闭环后系统自动归档 → 归档后不可编辑，仅可查看
  支持按多条件筛选查看 + 导出
```

## 四、通用说明

1. **碳基守门 (Human-in-the-Loop)**: 每个AI决策节点均需人工审核确认后方可进入下一阶段
2. **方案变更 + 划词调整**: 所有AI输出物均支持两种编辑方式
3. **输出格式**: Word (docx)、Excel (xlsx) 等常用格式
4. **权限管理 (RBAC)**: 集团/科沃斯/添可 三级角色，行级数据过滤
5. **A2A通信**: 龟宝(HR)、西塞罗(法务)、波特(财务) 通过统一协议协作
6. **风控系统对接**: 特定节点调用智能体 → 守门后自动同步字段 → 人工补充 → 审批归档
7. **知识库**: 各模块独立知识库，离线本地维护，增量更新
