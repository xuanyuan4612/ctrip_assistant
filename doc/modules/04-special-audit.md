# 专项审计模块需求文档

## 一、当前流程

### 1.1 业务流程总览

专项审计的现行流程如下：

```
风控系统立项 → 制定项目方案 → 发送审计通知 → 访谈阶段 → 作业阶段
→ 输出初版缺陷清单 → 与被审计单位沟通缺陷情况 → 确认终版缺陷清单
→ 输出专项审计报告 → 各业务部门跟进整改
```

### 1.2 具体执行步骤

| 步骤 | 说明 |
|------|------|
| 风控系统立项 | 在风控系统中录入审计项目信息 |
| 制定项目方案 | 根据立项信息人工编制审计项目方案 |
| 发送审计通知 | 立项完成后发送审计通知，按审计方案开展专项审计工作 |
| 访谈阶段 | 确定访谈人员、编制访谈问卷、实施访谈、补充收集资料 |
| 作业阶段 | 收集资料、实施审计作业 |
| 输出初版缺陷清单 | 根据审计发现整理并输出初版缺陷清单 |
| 与被审计单位沟通缺陷情况 | 将初版缺陷清单发送至被审计单位进行沟通确认 |
| 确认终版缺陷清单 | 结合反馈意见确认最终的缺陷清单 |
| 输出专项审计报告 | 基于终版缺陷清单出具正式的专项审计报告 |
| 各业务部门跟进整改 | 业务部门根据审计报告中的整改建议落实整改措施 |

---

## 二、业务痛点

### 2.1 方案编制依赖个人经验，缺乏知识沉淀

- 审计方案无统一的知识沉淀机制，均依靠审计人员个人业务经验进行方案编制。
- 不同审计人员的经验差异导致方案质量参差不齐，缺乏标准化输出。
- 历史项目的审计方案和审计发现无法被有效复用。

### 2.2 人工查询效率低下

- 传统方式依赖人工逐个查询审计相关信息，程序繁琐且耗时较长。
- 审计人员需要在多个系统间切换，手动收集数据，严重影响审计工作的整体效率和进度。

### 2.3 数据量大且多元，处理困难

- 审计涉及的业务数据、流程数据、行为数据来源多、体量大、格式不一。
- 人工处理海量数据容易遗漏关键信息，难以保证审计覆盖的完整性。

## 三、知识库需求

### 3.1 知识库内容清单

| 序号 | 知识库内容 | 说明 |
|------|-----------|------|
| 1 | 历史项目审计发现清单 | 过往所有专项审计项目输出的问题清单及审计发现明细 |
| 2 | 历史审计方案 | 已完成的专项审计方案文档，用于方案生成阶段的检索与复用 |
| 3 | 公司组织架构、花名册、岗位职责 | 用于访谈阶段匹配被访谈人员，确保问卷发送至正确的岗位负责人 |
| 4 | 各个业务循环所涉及的公司各系统数据 | 包括业务数据、流程数据、行为数据，覆盖各业务循环的全链路数据 |
| 5 | 访谈问卷模板 | 标准化的访谈问题模板库，按业务类型分类 |
| 6 | 历史改善建议 | 过往审计报告中提出的整改建议，用于新报告中的整改建议生成 |

### 3.2 数据平台集成

- 知识库需与数据中台打通，支持 AI 自动从数据中台获取业务数据、流程数据和行为数据。
- 保留业务资料手动上传的窗口，适配未全量接入中台的业务场景及新增业务场景。

---

## 四、智能体输出成果（4 项）

| 序号 | 输出成果 | 说明 |
|------|---------|------|
| 1 | 标准化审计方案 | 包含审计范围、审计目标、审计方法、审计程序的完整方案文档 |
| 2 | 审计工作底稿 | 半自动填充的审计工作底稿，包含检查过程、数据来源、分析结果 |
| 3 | 审计报告 | 包含草稿版和正式版，自动生成整改建议 |
| 4 | 审计问题清单 | 审计过程中发现的所有问题汇总，支持逐条下发确认 |

## 五、具体工作流程

整体工作流包含 5 个阶段，按顺序串行执行：

```
（一）审计方案生成
    ↓
（二）访谈作业
    ↓
（三）检查作业
    ↓
（四）问题确认
    ↓
（五）出具审计报告
```

### （一）审计方案生成

#### 1. 功能描述

AI 根据专项审计确认的审计目的、审计重点，结合历史方案库、历史审计记录，自动生成包含审计范围、调研方法、抽样策略、时间计划、人员规划的详细审计方案。

#### 2. 输入数据

| 数据项 | 数据来源 | 说明 |
|--------|---------|------|
| 审计目的 | 风控系统立项输入 | 本次审计的核心目标 |
| 审计重点 | 风控系统立项输入 | 重点关注的业务领域或环节 |
| 审计区间 | 风控系统立项输入 | 审计覆盖的时间范围（YYYY-MM-DD 至 YYYY-MM-DD） |
| 被审计单位 | 风控系统立项输入 | 被审计的业务部门或单位 |
| 项目组组长 | 风控系统立项输入 | 审计项目负责人 |
| 项目组组员 | 风控系统立项输入 | 审计项目组成员 |
| 审计方案库 | 知识库维护 | 以往的审计方案文档 |
| 审计记录库 | 知识库维护 | 每个审计方案实施后的检查记录及检查结果 |

#### 3. 匹配逻辑

##### 3.1 审计范围及审计程序生成逻辑

1. 在"审计方案库"中检索与本次审计目的、审计重点关联的历史方案。
2. 获取对应方案的历史检查记录。
3. 综合输出优化后的审计方案，覆盖审计范围、调研方法、审计程序。

##### 3.2 抽样方案生成逻辑

1. AI 模型根据业务量级评估样本总体规模。
2. AI 模型根据风险程度评估抽样覆盖率要求。
3. 综合建议样本总量和抽样方法。

#### 4. 输出内容

系统最终生成的《专项审计实施方案》应包含以下标准字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **审计目的** | 本次专项审计需达成的核心目标 | 评估采购流程合规性，识别潜在风险 |
| **审计期间** | 审计覆盖的时间范围（根据立项输入） | 2025-01-01 至 2025-12-31 |
| **审计重点** | 具体重点审计业务及职责范围 | 采购招标合规性、供应商准入管理 |
| **审计方法** | 针对每个审计重点列出的具体手段 | 数据分析、凭证抽查、人员访谈 |
| **抽样方案 - 样本总量** | AI 建议的具体抽样数值 | 建议抽查 50 笔 |
| **抽样方案 - 抽样策略** | AI 建议的抽样方法 | 大额交易 100% 覆盖，小额交易随机抽取 10% |

审计范围包含字段：分析对象、BU（业务单元）、事业部、相关描述、审计规则。

---

### （二）访谈作业

#### 1. 功能描述

基于审计方案中的重点审计业务判断是否需要访谈。若需要访谈，逐步生成访谈计划、访谈问卷、访谈结果。访谈结果回传至风控系统，并可被下一阶段（检查作业）调用，以完善资料清单。

#### 2. 输入内容

| 数据项 | 来源 / 说明 |
|--------|------------|
| 审计方案 | 上一节点生成的方案，提取重点审计业务 |
| 公司人员岗位职责 | 知识库，用于匹配被访谈人员 |
| 知识库：访谈问题库 | 标准化访谈问题模板 |
| 问题清单 | 非必选项，若检查作业在前则作为输入 |

#### 3. 输出内容

| 输出项 | 说明 |
|--------|------|
| 访谈计划 | 包含访谈清单和访谈策略 |
| 访谈问卷 | 针对各被访谈人员生成的个性化问卷 |
| 访谈结论 | 访谈结果的分析与总结 |

#### 4. 作业步骤

**步骤一：生成访谈计划**

1. 根据审计方案守门确认的需要访谈内容。
2. 调用人事 MCP（人员信息接口），根据重点审计业务匹配访谈人员。
3. 输出访谈计划，包含被访谈人员、访谈人员（审计方）、访谈时间、访谈策略。
4. 人工守门确认访谈计划。

**步骤二：生成并下发访谈问卷**

1. 根据访谈计划，针对各被访谈人员生成个性化的访谈问卷。
2. 人工守门确认问卷内容。
3. 通过 ATA（自动化任务分配系统）向各被访谈人员发送任务及访谈问卷。

**步骤三：访谈结果分析与处理**

1. 线下访谈完成后，现场直接上传访谈结果（文字记录）。
2. AI 分析访谈结论的完整性，判断是否需要额外补充提问。
3. 若需要补充提问，继续补充访谈。
4. 若无需补充提问，任务结束，输出待确认的访谈问卷，由被访谈方签字确认后上传至风控系统。

---

### （三）检查作业

#### 1. 功能描述

基于审计方案中的审计目的、审计重点，结合资料清单库、业务流程库，生成需获取的资料清单及检查计划。资料涵盖内部资料（业务数据 + 行为数据）和外部资料。

#### 2. 输入内容

| 数据项 | 来源 / 说明 |
|--------|------------|
| 审计方案 | 上一节点生成的方案，提取重点审计业务、审计目的 |
| 知识库：标准资料需求模板库 | 按业务类型分类的资料清单模板 |
| 访谈结果 | 非必选项，若访谈在前则作为输入 |
| 资料检查结果 | 非必选项，由 AI 判断资料是否完整，若不完整则补充执行 |

#### 3. 输出内容

| 输出项 | 说明 |
|--------|------|
| 资料检查计划 | 包含所需获取的资料清单、分析维度、检查重点 |
| 审计底稿 | 半自动填充的审计工作底稿，记录检查过程与结果 |
| 问题清单 | 检查过程中发现的审计问题汇总 |

#### 4. 作业步骤

**步骤一：生成资料检查计划**

1. 根据审计方案守门确认的业务重点。
2. 结合资料清单模板库、业务流程库等知识库内容。
3. 输出完备的资料检查计划，涵盖所需获取的资料清单、分析维度、检查重点。
4. 人工守门确认资料检查计划。

**步骤二：执行检查与数据获取**

1. 根据资料清单和分析维度，AI 自动生成 SQL 语句从数据中台获取业务数据。
2. 需要从外部获取资料的部分，AI 调用 API 接口获取相关资料。
3. AI 根据检查点逐一检查是否存在问题。
4. 输出数据审计底稿和问题清单。
5. 保留业务资料手动上传的窗口，适配以下场景：
   - 当前资料未全量同步至数据中台的情况。
   - 业务场景新增导致资料类型增加的情况。

**SQL 自动生成说明**：

- AI 根据检查维度自动生成对应的 SQL 查询语句，目标为数据中台。
- SQL 涵盖数据筛选、聚合、异常值识别、关联比对等操作。
- 生成的 SQL 需经过审计人员确认后执行，确保数据获取的准确性。
- 支持对 SQL 查询结果进行二次分析和可视化呈现。

**步骤三：汇总分析与方向判断**

1. 各项资料获取和检查完毕后，AI 汇总分析是否达成审计目的。
2. 判断是否可以得出审计结论。
3. 若无法达成审计目的，AI 提出资料补充获取的建议及下一步审计方向。

---

### （四）问题确认

#### 1. 功能描述

上一步完成问题清单后，通过 ATA 将各项问题下发至任务中心（每条问题一条独立任务），由岗位负责人确认问题是否属实并反馈。

#### 2. 输入内容

| 数据项 | 来源 |
|--------|------|
| 审计发现（问题清单） | 检查作业阶段的输出 |

#### 3. 输出内容

| 输出项 | 说明 |
|--------|------|
| 经确认的审计问题 | 经过被审计方确认或 AI 分析判定后的最终问题清单 |

#### 4. 作业步骤

**步骤一：问题下发**

1. 将问题清单按问题逐条拆分。
2. 通过 ATA 将每条问题作为独立任务下发至任务中心，指定对应岗位负责人。

**步骤二：反馈处理与 AI 分析**

1. 岗位负责人接收任务后反馈确认或不确认。
2. **若反馈确认**：该问题成立，直接进入下一步（出具审计报告阶段）。
3. **若反馈不确认**：
   - AI 根据业务流程、审计发现、公司制度等维度，分析被审计方的反馈是否合理。
   - **反馈合理**：关闭该问题，将反馈记录保存至知识库，避免后续审计输出相同问题。
   - **反馈不合理**：AI 输出理由说明，将任务继续下发至任务中心要求重新反馈。

---

### （五）出具审计报告

#### 1. 功能描述

待审计发现问题确认完毕后，AI 生成整改建议并出具专项审计报告。

#### 2. 输入内容

| 数据项 | 来源 |
|--------|------|
| 项目基本资料 | 风控系统立项信息及全过程审计资料 |
| 问题清单 | 经确认的最终版审计问题清单 |

#### 3. 输出内容

| 输出项 | 说明 |
|--------|------|
| **整改建议** | 针对每个审计问题生成的整改建议，参考历史改善建议知识库 |
| **审计报告** | 完整的专项审计报告，包含草稿版和正式版 |

#### 4. 作业步骤

1. AI 基于确认后的审计问题清单，结合历史改善建议知识库，为每个问题生成整改建议。
2. AI 汇总项目全过程的审计信息，生成专项审计报告草稿。
3. 审计人员审核并修订报告草稿，输出正式版专项审计报告。
4. 各业务部门根据审计报告中的整改建议跟进落实整改措施。

**审计报告应包含以下内容**：

- 审计项目基本信息（项目名称、审计期间、被审计单位、项目组成员）。
- 审计目的与审计范围。
- 审计方法及审计程序说明。
- 审计发现及问题清单（含整改建议）。
- 审计结论与总体评价。

## 六、附录

### 6.1 各阶段输入输出汇总

| 阶段 | 输入 | 输出 |
|------|------|------|
| 审计方案生成 | 审计目的/重点/区间、被审计单位、项目组成员、历史方案库、审计记录库 | 专项审计实施方案 |
| 访谈作业 | 审计方案、人员岗位职责、访谈问题库、问题清单（可选） | 访谈计划、访谈问卷、访谈结论 |
| 检查作业 | 审计方案、资料模板库、访谈结果（可选）、资料检查结果（可选） | 资料检查计划、审计底稿、问题清单 |
| 问题确认 | 审计发现（问题清单） | 经确认的审计问题 |
| 出具审计报告 | 项目基本资料、问题清单 | 整改建议、审计报告 |

### 6.2 涉及的关键系统与接口

| 系统 / 接口 | 用途 |
|------------|------|
| 风控系统 | 立项输入、任务下发、资料存储 |
| 数据中台 | 业务数据、流程数据、行为数据的存储与查询 |
| 人事 MCP | 人员信息查询、岗位职责匹配 |
| ATA（自动化任务分配系统） | 访谈问卷下发、问题确认任务下发 |
| 知识库 | 存储历史方案、审计记录、访谈模板、资料模板、改善建议 |
