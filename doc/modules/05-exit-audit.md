# 离任审计模块需求说明书

> 文档版本：V2.0
> 所属系统：科沃斯-赫尔墨斯AI系统
> 模块名称：离任审计

---

## 1. 当前流程（BPM/OA审批同步）

### 1.1 现有流程概览

离任审计当前采用线下人工驱动的工作流程，整体链路如下：

```
风控系统立项（BPM/OA审批完成后同步）
    ↓
制定项目方案（人工编制）
    ↓
发送审计通知
    ↓
访谈阶段（确定访谈人员、编制访谈问卷、访谈实施、资料收集补充）
    ↓
作业阶段（收集资料、实施审计作业）
    ↓
输出初版缺陷清单
    ↓
与被审计单位沟通缺陷情况
    ↓
确认终版缺陷清单
    ↓
输出个人离任审计报告和离任审计总报告
    ↓
各业务部门跟进整改
```

### 1.2 流程触发条件

离任审计流程由风控系统立项触发。立项审批在 BPM/OA 系统中完成后，同步数据至风控系统，风控系统根据审计费方案启动离任审计工作。

---

## 2. 业务痛点

| 痛点 | 描述 |
|------|------|
| **审计方案无统一知识沉淀** | 审计方案完全依赖个人业务经验编制，缺少标准化知识库支撑，经验无法传承复用 |
| **人工查询整理效率低下** | 传统依赖人工逐个查询审计相关信息，程序繁琐且耗时较长，影响审计工作的整体效率和进度 |
| **数据量大且多元造成处理困难** | 审计涉及多业务系统数据、外部数据、人工数据等多源异构数据，人工处理难以有效整合分析 |

---

## 3. 知识库与数据需求

### 3.1 知识库清单

| 知识库 | 说明 | 用途 |
|--------|------|------|
| 历史项目审计发现清单 | 过往离任审计项目中发现的问题记录 | 问题清单生成、审计参考 |
| 历史审计方案清单 | 历史离任审计方案的完整记录 | 审计方案生成时匹配参考 |
| 公司组织架构、花名册、岗位职责 | 公司各部门架构、人员信息、岗位职责定义 | 审计期间确认、职责匹配 |
| 各业务系统数据 | 各个业务循环所涉及的公司各系统数据（业务数据 + 流程数据） | 问题清单生成阶段数据验证分析 |
| 方案、访谈清单、报告模板 | 审计方案模板、访谈问卷模板、审计报告模板 | 各阶段输出格式化 |
| 历史改善建议 | 过往审计项目中提出的改善建议及整改情况 | 审计报告建议参考 |
| 审计方案库（个人审计方案库 + 业务审计方案库） | 针对个人审查的通用审计策略，以及针对业务流程的标准审计程序 | 审计方案生成匹配 |
| 访谈问题库 | 按业务类型分类的通用访谈问题 | 访谈问卷生成 |
| 标准资料需求模板库 | 按业务类型分类的标准资料需求模板 | 资料清单生成 |
| 审计报告模板 | 标准格式的离任审计报告书模板 | 审计报告生成 |

### 3.2 外部数据

| 数据类别 | 具体内容 | 用途 |
|----------|----------|------|
| 天眼查外部数据 | 企业工商信息、关联关系 | 问题清单生成阶段识别关联交易、关联公司风险 |
| 行为风险预警数据 | 赫尔墨斯AI其他智能体模块传回的风险预警结果 | 问题清单生成阶段风险评估 |

## 5. 具体工作流程

离任审计工作流由 6 个阶段构成，按以下顺序执行：

```
（一）审计方案生成
    ↓
（二）生成访谈问卷
    ↓
（三）生成资料清单
    ↓
（四）问题清单生成
    ↓
（五）审计问题确认
    ↓
（六）出具审计报告
```

---

### （一）审计方案生成

#### 功能描述

系统根据被审计人的"职责配置"和"任职时间"，结合"方案库"中的历史模板与 AI 算法，自动生成包含审计范围、方法、样本策略及时间计划的详细审计方案。

#### 输入数据

审计方案生成需要调用以下三个数据模块：

##### （1）被审计人基础信息

| 数据项 | 说明 |
|--------|------|
| 输入来源 | 风控系统立项审批结束后推送并触发 |
| 事业部 | 被审计人所属事业部 |
| 离职人姓名 | 被审计人姓名 |
| 离职人工号 | 被审计人工号 |
| 离职人岗位 | 被审计人岗位名称 |
| 离职人部门 | 被审计人所属部门 |
| 入职日期 | 被审计人入职日期 |
| 最后工作日 | 被审计人最后工作日（离职日期） |
| 组长名称 | 审计项目组长 |
| 组员名称 | 审计项目组员 |
| 项目名称 | 审计项目名称 |
| 项目编号 | 审计项目编号 |
| 审计期间 | 系统自动计算的审计覆盖时间段 |

##### （2）职责配置库

| 数据项 | 说明 |
|--------|------|
| 来源 | MCP 数据调用 |
| 内容 | MCP 中该人员对应的具体职责内容 |

##### （3）审计方案库

| 数据项 | 说明 |
|--------|------|
| 来源 | 知识库维护 |
| 个人审计方案库 | 针对个人审查的通用审计策略 |
| 业务审计方案库 | 针对涉及到的业务流程的标准审计程序 |

#### 匹配逻辑

##### 审计范围及审计程序生成逻辑

| 步骤 | 动作 |
|:----:|------|
| 1 | 系统读取被审计人的岗位职责列表 |
| 2 | 在"业务审计方案库"中检索与这些职责强关联的业务内容，并获取对应业务的审计检查办法 |
| 3 | 在"个人审计方案库"中检索该岗位通用的以及特有的检查项 |

##### 审计期间确认逻辑（自动计算规则）

系统根据被审计人的任职时间，按以下规则自动计算审计期间：

| 任职时间条件 | 审计期间计算规则 |
|-------------|----------------|
| 1 年 < 本岗位时间 <= 5 年 | 审计期限 = 离职日期向前计算 **三年** |
| 本岗位时间 > 5 年 | 审计期限 = 离职日期向前计算 **一年** |
| 本岗位时间 <= 1 年 | 按实际任职期间覆盖（默认规则） |

> **计算示例**：若某被审计人在本岗位任职 3 年，离职日期为 2025-06-30，则审计期间为 2022-07-01 至 2025-06-30（前三年）。若任职 8 年，则审计期间为 2024-07-01 至 2025-06-30（前一年）。

##### 抽样方案生成逻辑

| 步骤 | 动作 |
|:----:|------|
| 1 | AI 模型根据业务量级评估样本规模 |
| 2 | AI 模型根据风险程度评估抽样策略 |
| 3 | 输出建议的样本量和抽样方法 |

#### 输出内容

系统最终生成的《离任审计实施方案》应包含以下标准字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| （1）被审计人信息 | 姓名、岗位、部门 | 张三、采购经理、采购部 |
| （2）审计期间 | YYYY-MM-DD 至 YYYY-MM-DD | 2022-07-01 至 2025-06-30 |
| （3）审计范围 | 职责范围 + 业务范围 | 职责：采购管理、供应商管理；业务：采购业务、费用报销 |
| （4）审计方法 | 针对每个业务范围列出的具体手段 | 数据分析、凭证抽查、人员访谈 |
| （5）抽样方案 | 样本总量 + 抽样策略 | 建议抽查 50 笔；大额交易 100% 覆盖，小额交易随机抽取 10% |

### （二）生成访谈问卷

#### 功能描述

基于审计方案中的重点审计领域（如高风险采购、大额销售）以及被审计人的岗位职责，自动生成一份结构化、定制化的访谈提纲。

#### 输入内容

| 输入项 | 说明 | 来源 |
|--------|------|------|
| （1）审计方案 | 提取审计范围（如：采购业务、费用报销）和重点关注领域（如：单一来源采购占比过高） | 上一阶段（一）产出 |
| （2）被审计人岗位职责 | 被审计人的具体岗位职责描述 | 职责配置库（MCP） |
| （3）访谈问题库 | 按业务类型分类的通用访谈问题 | 知识库 |

#### 输出内容

| 输出项 | 说明 |
|--------|------|
| 结构化《离任审计访谈问卷》 | 包含基于审计方案和岗位职责定制的访谈问题列表 |

#### 访谈问卷确认（新增需求）

访谈问卷生成后，需增加以下确认流程：

| 功能 | 说明 |
|------|------|
| **离职人是否可确认** | 系统提供"是否"选择按钮。选择"是"，则直接发送给离职人员任务中心进行确认 |
| **审计人员确认（下拉框选择）** | 选择"否"，则从本项目的组长和组员中以下拉框形式选择审计人员，由审计人员确认问卷 |
| **被访谈人补充** | 生成问卷时需补充填写被访谈人信息 |

---

### （三）生成资料清单

#### 功能描述

基于审计方案中的审计范围，自动生成一份资料需求清单。

#### 输入内容

| 输入项 | 说明 | 来源 |
|--------|------|------|
| （1）审计方案 | 提取其中的审计范围 | 上一阶段（一）产出 |
| （2）标准资料需求模板库 | 按业务类型分类的资料需求模板 | 知识库 |

#### 输出内容

| 输出项 | 说明 |
|--------|------|
| 资料需求清单 | 审计所需资料的详细列表 |

#### 与任务中心交互（新增需求）

资料清单生成后，需增加以下功能：

| 功能 | 说明 |
|------|------|
| **资料提供人员名称** | 资料需求清单中增加"资料提供人员名称"字段，明确每项资料的提供责任人 |
| **同一人员多条资料合并发送** | 若多条资料由同一个人提供，则合并为一条任务发送至任务中心，避免同一人收到多条重复通知 |

---

### （四）问题清单生成

#### 功能描述

对业务数据、行为数据进行验证分析，识别存在的风险场景，生成结构化的审计问题清单。

#### 输入内容

| 输入项 | 说明 | 来源 |
|--------|------|------|
| （1）天眼查外部数据 | 企业工商信息、关联关系数据 | 外部数据接口 |
| （2）业务系统数据 | BPM/OA/DTC 等系统的日志与业务单据 | 各业务系统 |
| （3）人工上传线下数据 | 线下表格、截图等补充资料 | 审计人员手动上传 |
| （4）行为风险预警 | 智能体其他模块传回的风险预警结果 | 赫尔墨斯AI其他模块 |
| （5）历史审计问题 | 历史个人审计问题和业务审计问题参考 | 知识库 |

#### 输出内容

问题清单分为两大类：

##### 个人问题清单

涉及个人层面的风险问题，主要包括：

| 问题类别 | 说明 |
|----------|------|
| 商业秘密泄露 | 离职人员是否存在泄露公司商业秘密的行为 |
| 个人报销 | 个人报销是否存在违规、虚假报销等行为 |
| 样机使用 | 样机领用、使用、归还是否存在异常 |
| 关联公司 | 个人是否与关联公司存在不当利益往来 |
| 资产使用 | 个人使用的公司资产是否存在违规处置或侵占 |

##### 业务问题清单

涉及业务层面的风险问题，主要包括：

| 问题类别 | 说明 |
|----------|------|
| 流程漏洞 | 业务流程中存在的不规范、不完善的环节 |
| 制度缺陷 | 现行制度规定存在缺失、冲突或滞后的问题 |
| 经济损失 | 因管理不当或违规操作导致的公司经济损失 |

---

### （五）审计问题确认

#### 功能概述

将第四阶段生成的问题清单明细进行汇总，经风控人员守门确认后，与业务部门沟通确认，最终生成审计意见表推送至风控系统。

#### 处理流程

| 步骤 | 动作 | 说明 |
|:----:|------|------|
| 1 | **问题清单汇总** | 将第四模块发现的个人问题清单和业务问题清单进行汇总 |
| 2 | **风控人员守门确认** | 风控人员对问题清单进行审核确认，输出最终版缺陷清单 |
| 3 | **与业务部门初步沟通** | 将最终版缺陷清单与业务部门进行沟通确认（通过发布任务或邮件方式） |
| 4 | **生成审计意见表** | 根据已与业务确认的审计发现问题生成审计意见表 |

#### 审计意见表字段

| 字段 | 说明 |
|------|------|
| 审计发现问题 | 经确认的问题描述 |
| 审计建议 | 针对每个问题的改进建议 |
| 审计沟通对象 | 业务沟通对接人信息 |

#### 输出

| 输出项 | 说明 | 推送目标 |
|--------|------|----------|
| 审计意见表 | 含审计发现问题、审计建议、审计沟通对象 | 推送至风控系统审计作业模块 |

---

### （六）出具审计报告

#### 功能描述

基于已确认的审计发现问题和审计报告模板，出具标准化的离任审计报告书。

#### 输入内容

| 输入项 | 说明 | 来源 |
|--------|------|------|
| （1）问题清单 | 已确认的"个人问题清单"和"业务问题清单" | 上一阶段（五）产出 |
| （2）审计报告模板 | 标准格式的审计报告书模板 | 知识库 |

#### 输出内容

| 输出项 | 说明 | 去向 |
|--------|------|------|
| （1）标准化《离任审计报告书》 | 经风控人员守门后推送 | 风控系统审计报告模块 |
| （2）结构化问题列表 | 传递给下游智能体的结构化数据 | 整改跟踪智能体 |

#### 输出流程

```
已确认问题清单 + 审计报告模板
    ↓
AI 生成标准化《离任审计报告书》（草稿）
    ↓
风控人员守门确认
    ↓
推送至风控系统审计报告模块（正式版）
    ↓
同时传递结构化问题列表至"整改跟踪智能体"
    ↓
各业务部门跟进整改
```

## 6. 个人问题与业务问题分类汇总

| 维度 | 个人问题清单 | 业务问题清单 |
|------|-------------|-------------|
| **关注焦点** | 个人行为的合规性与廉洁性 | 业务流程、制度的健全性与有效性 |
| **典型问题** | 商业秘密泄露、个人报销违规、样机使用异常、关联公司利益往来、资产违规处置 | 流程漏洞、制度缺陷、经济损失 |
| **风险性质** | 个人层面的行为风险 | 组织层面的运营风险 |
| **数据来源** | 天眼查关联关系 + 业务系统个人行为数据 + 行为风险预警 | 业务系统流程数据 + 人工上传资料 |
| **整改责任主体** | 离职人员及直接上级 | 业务部门及流程责任人 |

## 7. 全流程数据流转总览

| 阶段 | 输入 | 处理逻辑 | 输出 |
|:----:|------|----------|------|
| （一）审计方案生成 | 被审计人基础信息（风控系统）+ 职责配置库（MCP）+ 审计方案库（知识库） | AI 匹配职责与方案映射 + 自动计算审计期间 + AI 建议抽样方案 | 《离任审计实施方案》 |
| （二）生成访谈问卷 | 审计方案 + 岗位职责 + 访谈问题库 | AI 基于审计重点领域和岗位职责生成结构化访谈提纲 | 结构化《离任审计访谈问卷》 |
| （三）生成资料清单 | 审计方案 + 标准资料需求模板库 | AI 根据审计范围匹配资料需求模板 | 资料需求清单（含资料提供人信息） |
| （四）问题清单生成 | 天眼查数据 + 业务系统数据 + 人工上传 + 行为风险预警 + 历史问题 | AI 多源数据交叉验证分析，识别风险场景 | 个人问题清单 + 业务问题清单 |
| （五）审计问题确认 | 问题清单汇总 | 风控人员守门 + 业务部门沟通确认 + 生成审计意见表 | 审计意见表（推送风控系统） |
| （六）出具审计报告 | 已确认问题清单 + 审计报告模板 | AI 标准化生成 + 风控人员守门 | 离任审计报告书（推送风控系统）+ 问题列表（传递整改跟踪智能体） |
