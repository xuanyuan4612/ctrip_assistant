# 行为风险模块需求文档

> 版本：v1.0  
> 所属系统：科沃斯-赫尔墨斯AI系统
> 模块名称：行为风险 (Behavioral Risk)
> 状态：初稿

---

## 一、当前流程

对各监管系统产生的行为数据进行定期分析，识别异常行为，提前预警处理。

---

## 二、业务痛点

行为日志和文件涉及的数据量巨大，异常分析需要结合多个来源的跨系统信息（例如文件传输、文件解密、人事档案、亲属信息、商业秘密信息等），缺乏高效的分析工具。

---

## 三、业务改善点

利用智能体分析各系统行为数据，识别异常的员工行为，实现跨系统数据整合与智能分析，提升行为风险识别的效率和准确性。

---

## 四、知识库/数据库需求

| 序号 | 来源 | 内容 | 用途说明 |
|------|------|------|----------|
| 1 | 行为风险管理系统 | 各监管系统行为数据 | 核心分析数据源（需与IT沟通数据对接） |
| 2 | 风控系统 | 员工生命周期信息（离职流程）、利益申报信息（包含历史档案） | 员工状态校验与利益冲突分析 |
| 3 | 风控系统 | 商业秘密信息、涉密文件清单 | 涉密行为关联分析 |
| 4 | HR系统/德鲁克 | 员工档案信息（邮箱、微信/QQ、手机号等） | 员工身份与联系方式核实 |
| 5 | MDM/德鲁克 | 员工部门架构信息 | 组织归属与权限范围校验 |
| 6 | 威科先行 | 外部法律法规、侵权诉讼案件信息 | 行为合规性外部基准 |
| 7 | 风控智能体 | 前期行为分析结果 | 历史分析结论回溯与趋势研判 |
| 8 | 德鲁克 | 员工岗位职责 | 行为与岗位职责匹配分析 |

---

## 五、智能体输出成果

| 输出成果 | 说明 |
|----------|------|
| 行为风险分析报告 | 对指定范围内员工行为数据进行分析，识别异常行为并出具报告 |
| 行为风险管理情况报告 | 定期（月度）对公司整体行为风险管理情况进行分析和总结 |

## 六、前台功能

### 6.1 行为风险分析

#### 6.1.1 功能概述

在辛顿平台开发行为风险分析功能，选择分析范围后，赫尔墨斯基于多系统行为数据进行综合分析，输出行为风险分析报告。

#### 6.1.2 范围选择

分析范围选择参数：

| 参数 | 说明 |
|------|------|
| 事业部 | 按事业部维度筛选 |
| 部门 | 按部门维度筛选 |
| 岗位 | 按岗位维度筛选 |
| 人员 | 按具体人员筛选 |
| 角色 | 按角色维度筛选 |
| 期间 | 分析的时间范围 |

#### 6.1.3 输出内容

输出行为风险分析报告，包括异常行为识别、风险等级评估、关联分析结论等。

### 6.2 历史分析结果

查看往期行为风险分析报告，支持按时间、范围等条件检索历史分析结果。

### 6.3 行为风险管理情况总结报告

#### 6.3.1 功能概述

赫尔墨斯定期（月度）生成公司整体行为风险管理情况报告。

#### 6.3.2 报告内容

| 维度 | 说明 |
|------|------|
| 监管系统覆盖范围 | 已接入的监管系统清单及覆盖情况 |
| 缺失范围 | 尚未覆盖的监管盲区 |
| 行为数据质量 | 各系统行为数据的完整性、及时性、准确性评估 |
| 高风险行为展示 | 高风险行为事件汇总与详情 |
| 高频行为问题展示 | 频发行为问题分类统计与趋势分析 |
| 优化方案和建议 | 基于分析结论的管理改进建议 |

#### 6.3.3 展示方式

展示报告清单，点击可查看单项报告详情。

#### 6.3.4 历史管理报告

查看往期管理报告，支持按时间检索历史管理报告。

## 七、前台页面落地方案

### 7.1 主要工作

1. 辛顿平台开发行为风险分析功能，输出分析结果
2. 定期输出行为风险管理情况报告

### 7.2 辛顿前台页面需求

| 功能模块 | 说明 |
|----------|------|
| 行为风险分析 | 范围选择（事业部、部门、岗位、人员、角色、期间），输出行为风险分析报告 |
| 历史分析结果 | 查看往期分析报告 |
| 行为风险管理情况总结报告 | 展示报告清单，点击查看单项报告；月度生成 |
| 历史管理报告 | 查看往期管理报告 |

### 7.3 风控系统前台页面需求

风控系统行为风险管理模块按照新版需求文档落地，与辛顿平台形成数据联动和页面跳转配合。
