## 二、产品概述

### 2.1 产品定位

审计持续改善智能体是面向审计岗的专属智能跟踪工具，核心聚焦审计过程中发现问题的整改承接、全程跟踪、闭环管控场景，通过固定表头规范数据展示、流程化推进整改、智能提醒、全程留痕等功能，实现审计整改全流程标准化管理，降低审计岗跟踪成本，确保所有审计问题可追溯、可核查、可闭环，适配企业内部审计合规管理需求。

### 2.2 目标用户

| 用户类型 | 职责说明 |
|----------|----------|
| **核心用户：审计岗人员** | 负责承接、录入、跟踪、统计审计问题整改，开展整改复核，输出跟踪报告 |
| **关联用户：整改责任人** | 推进审计问题整改落地，提交整改材料、更新整改进度 |
| **关联用户：管理层** | 查看整改跟踪进度、闭环情况及统计数据 |

### 2.3 核心价值

- **统一管理**：打造审计问题整改单一跟踪入口，整合所有审计项目发现的整改事项，避免分散管理遗漏
- **规范标准**：通过固定表头统一审计整改数据口径、复核、闭环流程，满足审计合规要求
- **效率提升**：自动化跟踪整改进度、智能提醒逾期事项，减少人工手动跟踪、统计的工作量，提升整改推进效率
- **闭环管控**：全程记录审计问题从录入、下发、整改、复核到闭环的全流程，确保整改过程可追溯、整改结果可核查

##### 3.2.1.2 录入方式

| 录入方式 | 说明 |
|----------|------|
| **风控系统自动抓取** | 自赫尔墨斯（内控评价 / 廉洁监察 / 商业秘密 / 行为风险）模块风控系统整改跟踪模块自动抓取问题清单 |
| **手动单条录入** | 风控系统整改跟踪模块支持手动单条录入问题 |
| **Excel 批量导入** | 按固定表头模板下载填写后导入，导入后可手动编辑修改 |
| **问题作废** | 支持问题作废操作，需填写作废原因，留痕备查 |

**数据来源后处理流程**：数据来源后给组长发任务中心通知 → 组长确认并分配审计跟进人员。

##### 3.2.1.3 校验规则

| 校验项 | 规则说明 |
|--------|----------|
| **必填字段** | 问题编号、问题描述、责任部门、责任人、整改期限、问题类型、风险等级不能为空 |
| **问题编号唯一性** | 系统自动生成或手动录入后校验唯一性，不允许重复 |
| **整改期限校验** | 整改期限不能早于录入日期 |
| **风险等级** | 可按预设选项（高/中/低）选择 |

##### 3.2.1.4 状态说明

问题清单录入后自动在任务平台给审计组长创建任务"问题推送任务"，整改问题状态为"问题待推送"。全流程共设 **6 个状态**，流转关系如下：

```
问题待推送 → 计划待提交 → 计划待审批 → 整改答复待提交 → 已整改待复核 → 整改完成
```

| 状态 | 含义 |
|------|------|
| **问题待推送** | 审计问题清单同步至整改智能体后（组长确认下发前） |
| **计划待提交** | 审计组长下发审计整改任务后，整改部门还未上传审计整改计划/方案 |
| **计划待审批** | 整改部门上传审计整改计划/方案，审计跟进人员还未审核 |
| **整改答复待提交** | 审计人员已对整改部门上传的审计计划/方案审核完毕，整改部门还未上传最终整改数据 |
| **已整改待复核** | 整改部门已上传最终整改证据，审计跟进人员还未审核 |
| **整改完成** | 审计跟进人员已完成整改证据的审核，并确认通过 |

- 状态
- 事业部
- 项目年度
- 审计发现编号
- 审计项目编号
- 审计项目名称
- 项目组长
- 风控跟进人
- 是否超期

#### 3.2.2 整改任务下发功能

##### 3.2.2.1 权限视图

| 角色 | 可见范围 |
|------|----------|
| **项目组长** | 可看到负责项目下所有审计问题清单 |
| **项目成员** | 仅能看到被分配跟进的问题清单 |
| **事业部风控负责人** | 可看到该事业部所有问题清单 |
| **风控负责人** | 可看到所有问题清单 |

##### 3.2.2.2 下发流程

```
项目组长确认问题录入无误
        │
        ▼
一键"下发整改任务"
（"问题推送任务"自动完成）
        │
        ├─────────────────────────────────────┐
        ▼                                     ▼
系统推送整改任务至                    系统推送"计划/方案
对应部门负责人（任务中心）            审核任务"给审计跟进人
        │                                     │
        ▼                                     │
整改状态更新为                                    │
"计划待提交"                                     │
部门负责人收到Elink通知                          │
        │                                     │
        ▼                                     │
部门负责人在任务中心                                │
提交具体责任整改人                                │
提交审计整改计划/方案                              │
        │                                     │
        ▼                                     ▼
整改状态更新为 ←────────────────────────────
"计划待审核"
        │
        ▼
   AI初审：AI对上传附件进行初审并给出意见
        │
        ▼
   审计整改跟进人二次确认
        │
   ┌────┴────┐
   ▼         ▼
 通过       不通过 → 退回填写（备注原因）
   │                状态更新为"计划待提交"
   ▼
   审计计划/方案确认通过
   （审计跟进人计划审核任务自动完成）
        │
        ▼
   赫尔墨斯向业务部门推送"审计整改任务"
   （AI分析整改计划/方案，自动生成整改目标和任务描述）
        │
        ▼
   整改状态更新为"整改答复待提交"
   责任人收到Elink通知，明确整改要求及期限
```

**关键说明**：

1. 项目组长在赫尔墨斯确认问题录入无误后，可一键"下发整改任务"
2. 组长点击下发后，"问题推送任务"自动完成
3. 系统自动将整改任务推送至对应部门负责人（任务中心）
4. 同时自动推送给审计跟进人"计划/方案审核任务"
5. 下发后，整改状态自动更新为"计划待提交"，部门负责人可实时收到系统推送的任务待办通知（Elink通知）
6. 部门负责人在任务中心提交具体责任整改人、审计整改计划/方案等
7. 负责人提交后，整改状态更新为"计划待审核"
8. **AI 初审**：AI 对业务部门负责人上传的附件进行初审并给出意见
9. 审计整改跟进人进行二次确认
10. AI 与审计整改跟进人均确认通过后进入下一步
11. 若 AI 判断不通过或审计跟进人判断不通过，则退回重新填写，退回时需备注具体原因，整改状态更新为"计划待提交"
12. 审计计划/方案确认通过后（审计跟进人的计划审核任务自动完成）
13. 赫尔墨斯再向业务部门推送"审计整改任务"。该任务需与上述提交计划任务关联，AI 分析整改部门提交的整改计划/方案，自动生成整改目标和任务描述
14. 下发后，整改状态自动更新为"整改答复待提交"，责任人可实时收到系统推送的待办通知（Elink通知），并明确整改要求及期限

#### 3.2.4 整改复核与闭环功能

##### 3.2.4.1 复核全流程

```
责任人提交整改完成
        │
        ▼
状态更新为"已整改待复核"
        │
        ▼
   AI复核：AI依据前期上传的整改计划/方案
           对责任人上传的附件进行初审判断
        │
   ┌────┴────┐
   ▼         ▼
AI通过     AI不通过 → 填写退回原因
   │                 状态更新为"退回重改（AI退回）"
   │                 向责任人发送退回提醒
   ▼
AI初审核通过后
自动向审计岗Elink发送复核提醒
向审计岗任务平台推送复核任务
（明确待复核任务目标及描述）
        │
        ▼
   审计岗复核：查看整改说明、审核整改佐证材料
        │
   ┌────┴────┐
   ▼         ▼
复核通过   复核不通过 → 填写退回原因
   │                 状态更新为"退回重改（审计退回）"
   ▼                 向责任人发送退回提醒
状态更新为
"整改完成"
   │
   ▼
任务平台复核任务自动完成
   │
   ▼
自动归档（闭环）
```

##### 3.2.4.2 复核操作

- **AI 复核**：责任人提交整改完成（状态更新为"已整改待复核"）后，AI 依据前期上传的整改计划/方案，对责任人上传的附件进行初审判断
- **审计复核触发**：AI 初审核通过后自动向审计岗 Elink 发送复核提醒，并向审计岗任务平台推送复核任务，明确待复核任务目标及描述
- **复核操作**：审计岗查看整改说明、审核整改佐证材料，根据整改完成情况，选择"复核通过"或"复核不通过"

##### 3.2.4.3 闭环管理

| 场景 | 处理方式 |
|------|----------|
| **复核通过** | 任务平台复核任务自动完成，整改状态自动更新为"整改完成"，完成闭环 |
| **AI 复核不通过** | 填写退回原因（明确整改不足及补充要求），状态更新为"退回重改（AI退回）"，将任务退回责任人重新整改并再次新建任务，向责任人发送退回提醒 |
| **审计复核不通过** | 填写退回原因（明确整改不足及补充要求），状态更新为"退回重改（审计退回）"，将任务退回责任人重新整改并再次新建任务，向责任人发送退回提醒 |
| **退回达 3 次及以上** | 需同步通知审计问题整改责任人所在的部门负责人 |

##### 3.2.4.4 批量复核

可增加筛选功能，对不同类型的问题一键复核完成。例如一般问题经 AI 复核没问题后，复核人员可一键通过。

##### 3.2.4.5 复核留痕

所有复核操作（复核人、复核时间、复核意见、退回原因）永久留痕，不可篡改，满足审计合规核查需求。

##### 3.2.4.6 挽损及人员信息录入

风控跟进人、表头挽损情况（直接挽损金额、间接挽损金额）、人员处理情况开放审计人员手工录入，若发生信息变更，相关操作记录需同步保存。

#### 3.2.5 问题归档功能

##### 3.2.5.1 自动归档

整改闭环（复核通过）后，系统自动对该审计问题整改事项进行归档，归档后不可编辑，仅可查看。

##### 3.2.5.2 筛选查看

支持按以下条件筛选查看归档事项：

- 整改责任部门
- 责任部门负责人
- 责任部门跟进人
- 状态
- 事业部
- 项目年度
- 审计发现编号
- 审计项目编号
- 审计项目名称
- 项目组长
- 风控跟进人
- 是否超期

便于后续审计复盘、合规检查时调取查阅。

##### 3.2.5.3 数据导出

归档数据可按筛选条件导出，用于存档、复盘分析。

### 3.3 通用支撑模块

#### 3.3.1 统计分析功能

##### 3.3.1.1 自定义统计

支持按以下条件进行自定义统计：

- 时间段
- 审计项目
- 责任部门
- 问题类型
- 风险等级
- 整改状态

##### 3.3.1.2 统计指标

| 指标 | 说明 |
|------|------|
| 审计问题总数 | 统计范围内的全部审计问题数量 |
| 整改中数量 | 当前处于整改过程中的问题数量 |
| 已完成数量 | 已整改完成的问题数量 |
| 逾期数量 | 超过计划完成时间仍未完成的问题数量 |
| 退回重改数量 | 被退回要求重新整改的问题数量 |
| 整改完成率 | 已完成数量 / 审计问题总数 × 100% |
| 逾期率 | 逾期数量 / 审计问题总数 × 100% |

##### 3.3.1.3 报表生成

- 自动生成审计整改跟踪统计报表
- 支持 **Excel**、**PDF** 格式导出
- 报表内容包含统计指标、明细数据
- 可直接用于审计汇报、存档

##### 3.3.1.4 可视化展示

| 图表类型 | 展示内容 |
|----------|----------|
| **柱状图** | 各部门整改数量对比、各时间段整改进度对比 |
| **饼图** | 整改状态分布（整改中/已完成/逾期/退回重改占比） |
| **折线图** | 整改完成率/逾期率随时间变化趋势 |

所有图表支持导出，便于管理层查看、决策。

#### 3.3.2 操作日志功能

##### 3.3.2.1 全程留痕

所有操作均记录操作日志，包括但不限于：

- 问题导入
- 问题下发
- 进度更新
- 复核操作
- 退回操作
- 权限变更
- 数据导出

日志内容包含：**操作人**、**操作时间**、**操作内容**、**操作结果**。

##### 3.3.2.2 日志查询

- 支持按操作人、操作时间、操作类型筛选查询操作日志
- 日志不可删除、不可篡改，满足审计合规核查需求

##### 3.3.2.3 异常追溯

当出现数据错误、整改异常时，可通过操作日志追溯问题原因，明确责任主体。

## 四、交互需求

### 4.1 整体交互原则

简洁、高效、规范，适配审计岗日常操作习惯，重点优化问题录入、批量导入、责任人上传资料、AI复核/审计复核、统计导出的操作流程，减少操作步骤；页面布局清晰，聚焦审计整改核心功能，便于审计人员快速上手，同时满足整改责任人的简单操作需求。

### 4.2 核心交互流程

#### 4.2.1 审计问题整改全流程

```
自风控系统接入问题清单 / 赫尔墨斯导入审计问题
        │
        ▼
确认无误后下发整改任务
        │
        ▼
责任人收到提醒并更新整改进度 / 上传材料
        │
        ▼
责任人提交整改完成
        │
        ▼
AI 进行初审
        │
   ┌────┴────┐
   ▼         ▼
AI通过    AI不通过 ──→ 退回责任人重新整改
   │                      │
   ▼                      ▼
审计岗收到复核提醒        责任人重新提交
并开展复核                 （循环）
   │
   ▼
复核通过则闭环归档
复核不通过则退回责任人重新整改（循环）
```

#### 4.2.2 批量导入流程

```
审计岗下载固定表头模板
        │
        ▼
按模板填写数据
        │
        ▼
上传模板
        │
        ▼
系统校验数据
        │
   ┌────┴────┐
   ▼         ▼
校验通过   校验失败
   │         │
   ▼         ▼
完成导入   提示具体失败原因
           （如必填字段缺失、问题编号重复）
           在风控系统完成校验提示
```

### 4.3 关键页面交互要求

#### 4.3.1 问题录入页

- 固定表头字段按逻辑顺序排列（审计项目 → 问题编号 → 问题描述等）
- 必填字段标注红色星号（*）
- 支持字段提示（如问题类型、风险等级的选项说明）
- 提交前自动校验必填项

#### 4.3.2 批量导入页

- 提供固定表头模板下载入口
- 上传模板后显示导入进度
- 导入失败时明确提示失败条数及原因（如"问题编号重复"、"整改期限格式错误"等）
- 支持重新上传修改后的模板

#### 4.3.3 整改复核页

- 清晰展示整改事项明细、整改说明、整改佐证材料
- 复核意见填写区简洁明了
- 支持快速选择"复核通过" / "复核不通过"
- 退回时需强制填写退回原因（明确整改不足及补充要求）

#### 4.3.4 统计报表页

- 筛选条件可快速勾选
- 支持一键切换图表 / 表格展示
- 导出按钮显眼易操作
- 报表导出后保留固定格式，无需手动调整
